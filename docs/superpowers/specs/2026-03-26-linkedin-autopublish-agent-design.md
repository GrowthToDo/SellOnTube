# LinkedIn Autopublish Agent — Design Spec
**Date:** 2026-03-26
**Branch:** linkedin-bot
**Status:** Approved

---

## Overview

A weekly human-in-the-loop LinkedIn publishing system for SellonTube. Claude (via Claude Code) writes 5 LinkedIn posts each week based on SellonTube website content. A lightweight Node.js script schedules all 5 to the SellonTube LinkedIn company page via the Zernio API at 9 AM IST each weekday. Zernio handles the actual publishing automatically.

**Not built:** GitHub Actions, cron jobs, AI API keys, automated content discovery. Claude Code is the AI engine — invoked manually each week.

---

## Goals

- Publish one high-quality LinkedIn post every weekday at 9 AM IST
- Posts must be useful, specific, and relevant to B2B/SaaS founders evaluating YouTube for lead generation
- No repeated angles within 30 days
- Every post includes: source page link + image from the source page
- Minimal infrastructure — one script, one history file

---

## Weekly Workflow

```
1. User: "generate this week's LinkedIn posts"
2. Claude reads: SellonTube pages + linkedin-history.json
3. Claude writes: 5 posts (Mon–Fri, weekday themes, no repeated angles)
4. User reviews and approves (or requests edits)
5. User runs: node scripts/linkedin-schedule.js
6. Script POSTs all 5 to Zernio with scheduledFor dates at 9 AM IST
7. Zernio publishes automatically — done until next week
```

## Components

### 1. `queue.json` — Weekly post batch

Generated by Claude each week. Contains 5 post objects, one per weekday.

```json
[
  {
    "scheduledDate": "2026-03-30",
    "dayOfWeek": "Monday",
    "weekdayTheme": "Strategy",
    "sourceTitle": "The YouTube Acquisition Engine",
    "sourceUrl": "https://sellontube.com/blog/the-youtube-acquisition-engine",
    "imageUrl": "https://sellontube.com/images/og/youtube-acquisition-engine.jpg",
    "postAngle": "Why YouTube is a compounding acquisition asset, not a content calendar task",
    "linkedinPost": "Full post text here...",
    "hashtags": ["#YouTubeSEO", "#B2BMarketing"]
  }
]
```

### 2. `linkedin-schedule.js` — Zernio publishing script

Reads `queue.json`, POSTs each post to the Zernio API with a `scheduledFor` datetime at 9 AM IST. Appends successful publishes to `linkedin-history.json`.

**Zernio API call per post:**
```json
{
  "content": "<full post text with link>",
  "scheduledFor": "2026-03-30T03:30:00",
  "timezone": "Asia/Kolkata",
  "platforms": [
    {
      "platform": "linkedin",
      "accountId": "<ZERNIO_ACCOUNT_ID env var>"
    }
  ],
  "mediaUrls": ["<imageUrl if present>"]
}
```

- `scheduledFor` is always 9 AM IST = `T03:30:00Z` in UTC
- `mediaUrls` is omitted if `imageUrl` is null
- On success: prints confirmation + appends to history
- On failure: prints error with status code, does not crash — continues to next post

### 3. `linkedin-history.json` — Dedup log

Rolling log of the last 30 published posts. Claude reads this at the start of each weekly session to avoid repeating angles, hooks, or source pages.

```json
{
  "posts": [
    {
      "date": "2026-03-30",
      "dayOfWeek": "Monday",
      "weekdayTheme": "Strategy",
      "sourceTitle": "The YouTube Acquisition Engine",
      "sourceUrl": "https://sellontube.com/blog/the-youtube-acquisition-engine",
      "postAngle": "YouTube as compounding acquisition asset",
      "hook": "Most businesses treat YouTube like a content calendar task.",
      "hashtags": ["#YouTubeSEO", "#B2BMarketing"],
      "imageUrl": "https://sellontube.com/images/og/youtube-acquisition-engine.jpg",
      "status": "scheduled"
    }
  ]
}
```

## Content Generation Rules (Claude's weekly brief)

### Source pages (in priority order)
1. Blog posts (`/blog/`)
2. Tool pages (`/tools/`)
3. pSEO pages (`/youtube-for/`, `/youtube-vs/`)

### Weekday theme rotation
| Day | Theme | Topics |
|-----|-------|--------|
| Monday | Strategy | YouTube growth strategy, positioning, long-term leverage |
| Tuesday | SEO / Discoverability | YouTube SEO, search intent, keyword alignment |
| Wednesday | Lead Generation | YouTube for leads, buyer journeys, conversion |
| Thursday | Content / Messaging | Scripting, content planning, messaging clarity |
| Friday | Mistakes / Contrarian | What businesses get wrong, anti-patterns, myths |

### Post requirements (every post)
- Length: 900–1,700 characters
- Opening: strong hook — no "In today's digital landscape", no "Here are 5 tips"
- Link: exact source page URL embedded naturally in post text
- Image: og:image or featured image from the source page (passed as `imageUrl`)
- Hashtags: 0–3 max
- Tone: sharp, clear, business-aware — not motivational, not guru-ish
- No em dashes (site style guide rule)

### Dedup rule
Do not repeat the same angle or hook within the last 30 posts. If a source page was used recently, choose a different page or extract a completely different angle.

## Environment Variables

| Variable | Source | Purpose |
|----------|--------|---------|
| `ZERNIO_API_KEY` | GitHub Secret | Zernio Bearer token |
| `ZERNIO_ACCOUNT_ID` | GitHub Secret | LinkedIn company page account ID in Zernio |

For local runs: add both to a `.env` file in the project root (already gitignored).

---

## Error Handling

| Failure | Behaviour |
|---------|-----------|
| Zernio API returns non-2xx | Log error with status + body, skip that post, continue |
| `imageUrl` fetch fails | Publish without image (text + link only) |
| `queue.json` missing or empty | Exit with clear error message |
| Network timeout | Single retry after 3s, then log and skip |

---

## Out of Scope

- Automated content discovery (Claude reads pages manually each week)
- GitHub Actions / cron scheduling (Zernio handles scheduling)
- Comment/engagement management
- Analytics tracking
- Image generation (only use existing images from source pages)
